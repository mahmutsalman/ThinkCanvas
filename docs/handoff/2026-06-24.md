## What Was Tried and Abandoned
- **Auto-wrapping Go fragments to "just run."** Partially works, but Go's strictness
  (unused vars AND unused imports are hard errors) and cross-note references (e.g. an
  `Article` type defined in a *different* note) mean many illustration fragments cannot run
  in isolation. Concluded: structural wrap is worth it; chasing 100% Go runnability is not.
- **Mass-rewriting all 167 failing notes.** Rejected — most "failures" are by design:
  Java Greedy Top-75 are reference `class Solution` solutions (need a test harness, that's
  what Recall is for); Go-syntax board references external deps (`gin`, `pgxpool`); ~14
  Python notes are multi-example reference cards that were never a runnable sequence.

## Decisions Made
- **Execution lives in main, serialized.** Renderer never spawns; main owns a single FIFO
  queue (concurrency 1) — the RAM guarantee. Output is ephemeral (not saved to board JSON).
- **Auto-wrap fragments, run full programs literally** (detected by class/main/package).
- **Setup preamble over inline fixtures** — keeps memorized `code` pure; Recall still grades
  only `data.code`. (User chose this over inline-injecting setup into note bodies.)
- **TypeScript runs via Node type-stripping** (`--experimental-strip-types`); needs Node ≥22.18.
- **Boards are edited only while the app is quit** (autosave clobber); back up first.
