# Handoff — 2026-06-22 [session 1]

## Session Goal
Build a Scapple-style infinite canvas app from scratch: double-click to drop notes, drag-to-connect, Monaco code notes, dark theme — then iterate on window-drag and editable edges.

## What Got Done
- Scaffolded **ThinkCanvas** in `EfficiencApps3/` — Electron + React + React Flow + Monaco, mirroring FocusWriter2's electron-vite + offline Monaco worker setup.
- Canvas core (`App.tsx`): double-click empty pane → text note in edit mode; drag notes; **drag a note onto another → edge created and note snaps back** (pure connect gesture, no overlap); pan/zoom (two-finger pan, pinch zoom); dark dotted background; localStorage autosave + reload.
- Two node types: `TextNode` (borderless, type-to-edit, Enter commits, To-code toggle) and `CodeNode` (Monaco editor, language dropdown, resizable, header drag-handle).
- `FloatingEdge` — border-to-border dashed edge that re-anchors live as notes move (`floating-edge-utils.ts`, React Flow v12 geometry).
- Fixed: window was not draggable (canvas captured all mouse input) → added a dedicated `-webkit-app-region: drag` top bar **above** the canvas, with no-drag buttons.
- Edges made interactive: click selects (orange, 24px hit area), type-to-label / double-click-to-label, Enter saves, 11px label font (smaller than 14px notes), right-click → Edit/Remove context menu.
- Verified: `npm run build` clean, `tsc -p tsconfig.web.json --noEmit` clean, app runs via `npm run dev` (HMR live).

## What Was Tried and Abandoned
- **Approach**: title/tools as React Flow `<Panel>` overlays with `-webkit-app-region: drag`.
  - **Why we walked back**: panels sit inside the React Flow pane, which captures pointer events, so the window wouldn't drag.
  - **Lesson**: OS-draggable chrome must live outside the React Flow surface.

## Decisions Made
- ADR-001: Electron + React Flow + Monaco (see STATUS recent decisions; not yet a formal ADR file).
- Drag-onto-note connect snaps the note back to its drag-start position.
- Edge labels rendered via `EdgeLabelRenderer`; `BaseEdge` gives the clickable hit area.

## Open Questions for Next Session
- Explicit "clear label" vs. blank-disappears behavior for edges.
- File-based save/open instead of localStorage.
