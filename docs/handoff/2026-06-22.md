# Handoff — 2026-06-22 [session 1]

## Session Goal
Build a Scapple-style infinite canvas app from scratch: double-click to drop notes, drag-to-connect, Monaco code notes, dark theme — then iterate on window-drag and editable edges.

## What Got Done
- Scaffolded **ThinkCanvas** in `EfficiencApps3/` — Electron + React + React Flow + Monaco, mirroring FocusWriter2's electron-vite + offline Monaco worker setup.
- Canvas core (`App.tsx`): double-click empty pane → text note in edit mode; drag notes; **drag a note onto another → edge created and note snaps back** (pure connect gesture, no overlap); pan/zoom (two-finger pan, pinch zoom); dark dotted background; localStorage autosave + reload.
- Two node types: `TextNode` (borderless, type-to-edit, Enter commits, To-code toggle) and `CodeNode` (Monaco editor, language dropdown, resizable, header drag-handle).
- `FloatingEdge` — border-to-border dashed edge that re-anchors live as notes move (`floating-edge-utils.ts`, React Flow v12 geometry).
- Fixed: window was not draggable (canvas captured all mouse input) → added a dedicated `-webkit-app-region: drag` top bar **above** the canvas, with no-drag buttons.
- Edges made interactive: click selects (orange, 24px hit area), type-to-label / double-click-to-label, Enter saves, 11px label font (smaller than 14px notes), right-click → Edit/Remove context menu.
- Verified: `npm run build` clean, `tsc -p tsconfig.web.json --noEmit` clean, app runs via `npm run dev` (HMR live).

## What Was Tried and Abandoned
- **Approach**: title/tools as React Flow `<Panel>` overlays with `-webkit-app-region: drag`.
  - **Why we walked back**: panels sit inside the React Flow pane, which captures pointer events, so the window wouldn't drag.
  - **Lesson**: OS-draggable chrome must live outside the React Flow surface.

## Decisions Made
- ADR-001: Electron + React Flow + Monaco (see STATUS recent decisions; not yet a formal ADR file).
- Drag-onto-note connect snaps the note back to its drag-start position.
- Edge labels rendered via `EdgeLabelRenderer`; `BaseEdge` gives the clickable hit area.

## Open Questions for Next Session
- Explicit "clear label" vs. blank-disappears behavior for edges.
- File-based save/open instead of localStorage.

## Notes
- Standalone git repo initialized in `ThinkCanvas/` (separate from the AllLanguages vault).
- Run: `cd ThinkCanvas && npm run dev`. Package: `npm run build:mac`.
- More features requested by user immediately after this checkpoint — capture them next session.

# Handoff — 2026-06-22 [session 2, 12:23]

## Session Goal
Polish navigation feel and set sensible code-note defaults; then scope the next feature (MRU code-note cycler).

## What Got Done
- Java is now the default language for code notes (both the +Code button and text→code conversion); dropdown reordered Java, Python, JavaScript, C, TypeScript, Rust, C++, Go, SQL, JSON.
- Code notes shrunk from 360x220 to 240x200 (smaller, squarer).
- Click a note → camera smoothly glides it to center (`setCenter`, 450ms, current zoom); skips when the note is already selected so editing doesn't jerk the camera.
- Two-finger panning no longer stalls over notes: code notes only claim the wheel (`nowheel`) while selected, and when idle a transparent shield (`.tc-code__shield`) covers the editor so Monaco's native wheel listeners never fire — the wheel bubbles to the canvas and pans. Clicking the shield selects+centers the note (removing it).
- Cmd/Ctrl + wheel zooms the canvas anchored on the cursor, even while the cursor is inside an active editor — done via a native capture-phase wheel listener on the code body (a React handler can't stop Monaco's native listeners). Also set `zoomActivationKeyCode={['Meta','Control']}` so Cmd+wheel zooms over empty canvas too.
- Ignored `*.tsbuildinfo`.

## What Was Tried and Abandoned
- **Approach**: toggling only the `nowheel` CSS class to let panning pass over code notes.
  - **Why we walked back**: Monaco binds its own native wheel listeners; the class alone didn't stop it.
  - **Lesson**: to intercept Monaco's wheel you need a native capture-phase listener (or a non-Monaco shield element), not React props/classes.

## Decisions Made
- MRU code-note cycler design pending — recommendation is a circular MRU list (ring + cursor), not a destructive stack. Awaiting user confirmation on behavior before building.

## Open Questions for Next Session
- Cycler scope (code-only vs all), auto-add vs explicit pin, direction/reverse key, list cap, click-to-jump.

## Notes

# Handoff — 2026-06-22 [session 3, 13:05]

## Session Goal
Build the left-side MRU code-note cycler and the dive-in editing loop (cycle -> Enter -> type -> Esc).

## What Got Done
- Left-side MRU panel (`.tc-mru`): code notes only, auto-added to the top on click (deduped, capped 8, oldest evicted), each chip shows language badge + first code line; current chip highlighted; click a chip to jump. Auto-prunes deleted/converted notes.
- `.` cycles to the next/older code note, `Shift+.` reverses, wraps around; each step centers the camera (`setCenter`, 450ms) and selects the note. Skipped while typing (focus in input/textarea) or while an edge is selected.
- Enter dives into the selected code note's editor; Monaco keeps the cursor (nodes stay mounted) and we also restore it from `data.cursor` on mount and save it on blur, so it survives reloads.
- Esc bound inside Monaco blurs the editor (saving the cursor) so the canvas keys take over again — full mouse-free loop.
- New `src/renderer/src/lib/codeEditors.ts`: process-wide id->editor registry so App's Enter handler can focus the right editor.

## What Was Tried and Abandoned
- Considered a literal stack/queue for the cycler; rejected — stack `pop` is destructive (can't cycle back), queue gives oldest-first. Chose a circular MRU ring + cursor.

## Decisions Made
- Cursor saved on blur (not every move) — revisit if landing position ever feels stale.
- Cycler scope = code notes only; auto-track on click; cap 8.

## Open Questions for Next Session
- Cursor save cadence (blur vs move); chip remove/number-jump; file-based persistence.

## Notes
- Verified via `tsc -p tsconfig.web.json --noEmit` (clean) and live testing; clean restarts used after hook/onMount changes to avoid stale Fast Refresh.

# Handoff — 2026-06-22 [session 5, 14:17]

## Session Goal
Finish bug fixes from testing, then ship a real build: app icon, production package, dock install, line numbers.

## What Got Done
- Bug: Enter sometimes didn't focus the current code note. Cause = stale Monaco focus after clicking a different code note (React Flow selects it but doesn't move keyboard focus, so the old editor kept focus and the Enter handler's "is a textarea focused?" guard bailed). Fix = track current note via `activeCodeId` (set on cycle + click, cleared on text/pane click) and decide Enter with `editor.hasTextFocus()` on that note. Commit 6b6b530.
- Bug: transient `selected`/`dragging` persisted into board files → a note re-opened pre-selected, which suppressed the code-note pan-through shield and stalled panning over it. Fix = `stripTransient()` on save + load. Commit 6b6b530.
- App icon: orange connected node-graph on a dark squircle (resources/icon.svg → rsvg-convert → icon.png → sips/iconutil → icon.icns); wired `mac.icon` in package.json. Commit 48ab519.
- Production build: `npm run build:mac` → dist/mac-arm64/ThinkCanvas.app (icon embedded); copied to /Applications, cleared quarantine, pinned to dock via `defaults write com.apple.dock persistent-apps` + killall Dock.
- Line numbers in code notes (`lineNumbers:'on'`, `lineNumbersMinChars:2`); rebuilt + reinstalled the dock app. Commit 7ba1544.

## What Was Tried and Abandoned
- Screenshotting the app to inspect bugs — flaky (app on a second monitor / occluded); switched to inspecting on-disk board JSON, which is what actually pinned both bugs.

## Decisions Made
- Enter routing decoupled from React Flow selection (activeCodeId + hasTextFocus).
- Boards must never persist transient interaction state.
- Ship as an unsigned packaged .app in /Applications, pinned to dock.

## Open Questions for Next Session

- Optional code signing for sharing the app.
